## 一、项目背景

### 1.1 业务现状

玲音（铃音）项目正在推进大模型能力建设，涉及多个业务模块——老编辑、闻小妹、安全侠等。各模块分别承担不同方向的内容审核与质量评估任务，包括但不限于：

- **标题问题识别**（非资讯标题、地方新闻类、标题破碎、错别字病句等 8 个子维度）
- **广告宣传识别**（平台导流、引导售卖、宣传类等子维度）
- **非正规来源识别**（语料粗标工作进行中）
- 后续规划中的标题党、封面问题、内文问题、调性差、安全问题等方向

当前各任务的进度、评测结果、相关文档散落在不同的企微文档和 TAPD 需求单中，缺乏一个集中、直观的入口来了解整体推进情况。

### 1.2 核心痛点

| 痛点 | 具体表现 |
|------|----------|
| **进度不透明** | 10 个任务分别由不同人跟进，想了解整体完成率需要逐个询问或翻文档 |
| **评测结果分散** | 每个模型的准确率、召回率、F1 分数分散在各自的评测报告中，缺乏横向对比视角 |
| **文档找不到** | PRD、评测报告、分析文档等散落在企微文档和 TAPD 各处，新人或协作者难以快速定位 |
| **汇报效率低** | 每次项目会议或向上汇报，都需要临时整理数据和进度，重复劳动多 |

### 1.3 做这件事的初衷

> 想直观地看到铃音整体的、各个模块（老编辑、闻小妹、安全侠）的进度、完成情况和效果，并且关联各个任务的文档及 TAPD 需求。

一句话概括：**把散落各处的进度、效果、文档聚合到一块屏幕上，抬头就能看到全局。**

## 二、需求目标

### 2.1 核心目标

搭建一个**可投屏的进展大屏看板**，实现以下能力：

1. **一眼看进度**：总任务数、已完成、进行中、未开始，清清楚楚
2. **一眼看效果**：每个模型的准确率、召回率、F1 雷达图，一屏内完整呈现
3. **一键找文档**：每个团队成员关联的 PRD、评测报告、TAPD 需求链接，点击即达
4. **随时可分享**：通过公网链接分享给任何人，无需安装任何工具

### 2.2 使用场景

| 场景 | 说明 |
|------|------|
| **日常投屏** | 团队区域电视投屏，实时展示项目整体进展 |
| **项目周会** | 作为汇报素材，全屏展示进度和评测结果 |
| **跨团队协同** | 通过链接分享给协作方，无需反复同步进度 |
| **新人 onboarding** | 新加入的同事可以快速了解项目全貌和各模块状态 |

### 2.3 目标用户

- 项目负责人 / 产品经理
- 团队成员（老编辑、闻小妹、安全侠各模块负责人）
- 上级领导（汇报场景）
- 跨团队协作方

## 三、功能设计

### 3.1 整体架构

看板采用**多屏分页设计**，键盘左右箭头切换，F 键全屏，适配电视投屏。

```
┌─────────────────────────────────────────────────┐
│  顶部标题栏：项目名称 + 实时时钟 + 页面导航点     │
├─────────────────────────────────────────────────┤
│                                                 │
│              可切换的屏幕内容区                     │
│                                                 │
│  屏幕1：项目总览                                  │
│  屏幕2~N：各模型评测看板（按evaluations自动生成）   │
│  屏幕末：团队文档门户                              │
│                                                 │
├─────────────────────────────────────────────────┤
│  底部状态栏：数据加载状态 + 页面名称导航            │
└─────────────────────────────────────────────────┘
```

### 3.2 屏幕 1：项目总览

展示整体宏观信息，让人 3 秒内知道"项目进展到哪了"。

**包含模块：**

| 模块 | 内容 |
|------|------|
| **指标卡片（4 张）** | 总任务数、已完成数（含完成率）、进行中数、阻塞/未开始数 |
| **任务进展列表** | 各里程碑的名称、状态标签（✅/🔵/⚪）、目标日期、进度百分比条 |
| **本周要点** | 本周关键事项列表（人工维护） |
| **风险项** | 当前项目风险清单，按级别高亮显示 |

**当前数据示例：**
- 总任务数 10，已完成 2（完成率 20%），进行中 1，未开始 7
- 已完成：标题问题V1模型（2026-03-13）、广告&宣传V1模型（2026-03-20）
- 进行中：非正规来源第一批语料粗标（进度 18%，目标 2026-03-30）
- 未开始：标题党、封面问题、内文问题、调性差、安全问题

### 3.3 屏幕 2~N：模型评测看板（动态生成）

每个已上线模型独占一屏，展示评测效果的完整数据。屏幕数量根据 `evaluations` 数组长度自动计算——新增模型只需在数据中加一条记录，看板自动多出一屏。

**每屏包含：**

| 模块 | 内容 |
|------|------|
| **指标卡片（7 张）** | 当前模型名、F1 最高维度、待提升维度、降低进审量、准确率、召回率、上次评测日期 |
| **雷达图** | 各维度 F1 分数的能力雷达图（ECharts），支持当前版本 vs 上一版本对比 |
| **三级标签准召率表格** | 每个子维度的准确率、召回率、F1 分数明细 |

**当前已有模型：**

1. **标题问题 V1** — 8 个子维度（非资讯标题、地方新闻类、个人倾向性表述、标题序号、观点不明、错别字病句、影视剧标题、标题破碎），整体准确率 95%、召回率 46%
2. **广告宣传 V1** — 3 个子维度（平台导流、引导售卖、宣传类），尚未评测

**技术细节：**
- F1 计算公式：`F1 = 2 × Precision × Recall / (Precision + Recall)`
- 支持 `"--"` 占位符表示尚无数据（待评测状态），不会产生 NaN
- 雷达图支持版本对比，新版本用绿色实线，旧版本用灰色虚线

### 3.4 屏幕末：团队文档门户

汇总所有团队成员的任务状态和关联文档，形成"人 → 任务 → 文档"的关联视图。

**每个成员卡片包含：**
- 头像（取姓名最后一字）+ 姓名 + 角色
- 当前任务名称 + 状态标签 + 进度条
- 关联文档列表（点击跳转企微文档）
  - PRD（📋）
  - 评测报告（📊）
  - 技术文档（⚙️）
  - 其他文档（📄）

**后续规划：**
- 关联 TAPD 需求单链接，点击跳转到对应的需求/缺陷/迭代页面
- 按模块（老编辑/闻小妹/安全侠）分组展示

### 3.5 通用交互

| 交互 | 说明 |
|------|------|
| `← →` 键 | 切换页面 |
| `F` 键 | 进入/退出全屏 |
| 顶部导航点 | 点击跳转到指定页面 |
| 底部页面名称 | 点击跳转到指定页面 |
| 实时时钟 | 顶部右侧显示当前日期、星期、时分秒 |
| 数据自动刷新 | 每 5 分钟自动拉取最新数据 |

## 四、数据架构

### 4.1 数据源设计

采用**双数据源兜底机制**：

1. **外部 JSON 文件**（`data/dashboard.json`）—— 通过 HTTP 服务访问时优先加载
2. **HTML 内嵌数据**（`EMBEDDED_DATA`）—— 双击打开 HTML 时的兜底数据

更新数据时，两处需保持同步。

### 4.2 数据结构

```json

      "lastEvalDate": "2026-03-13",
```

### 4.3 扩展性设计

- **新增模型**：在 `evaluations` 数组末尾追加一条记录，看板自动新增一屏
- **新增成员**：在 `members` 数组追加成员卡片
- **新增里程碑**：在 `milestones` 数组追加任务节点
- **支持待评测状态**：`precision`/`recall` 可设为 `"--"` 字符串，页面优雅降级显示

## 五、技术方案

### 5.1 技术选型

| 维度 | 方案 | 理由 |
|------|------|------|
| **前端框架** | 纯原生 HTML + CSS + JavaScript | 单文件零依赖，双击即开，投屏友好 |
| **图表库** | ECharts 5.5（CDN 引入） | 雷达图表现力强，社区成熟 |
| **主题风格** | 深色大屏主题 | 投屏场景下更护眼、更有科技感 |
| **部署** | 本地 HTTP Server + SSH 隧道暴露公网 | 无需额外服务，快速分享 |

### 5.2 设计令牌体系

```
背景层级：#0b1120 → #111827 → #1a2332 → #1e293b
主色：蓝色系（#3b82f6）
语义色：绿色成功、黄色警告、红色错误
强调色：青色（#22d3ee）、紫色（#a855f7）、橙色（#fb923c）
字号：投屏友好（hero 3.5rem → body 1.05rem → caption 0.9rem）
圆角：12px（标准卡片）/ 8px（小组件）/ 16px（大容器）
```

### 5.3 响应式与适配

- 针对 1920×1080 电视屏幕优化
- 全屏模式下 body overflow: hidden，无滚动条
- 卡片入场动画（fadeInUp），逐个延迟 50ms
